## 📊 Enterprise Benchmark (Steam 1K Dataset)

To demonstrate TITAN-X's capabilities without exposing the proprietary core engine, we processed a dirty Steam dataset (1,000 corporate records containing high-density duplicates, varying string lengths, and noisy attributes).

### 🛠️ Execution Environment
The pipeline was benchmarked on a dual-GPU production setup to test distributed inference stability:
- **Compute:** $2 \times$ NVIDIA Tesla T4 (14.6 GB VRAM each)
- **Precision:** FP16 Hybrid Mode enabled
- **Topology:** Optimized Sparse Matrix via CSR

### 📈 Production Artifacts & Verifiable Results
You can inspect the exact outputs generated by TITAN-X in the `/benchmarks` directory:
- **`dedup_report.html`**: Interactive executive dashboard showing real-time metrics, entity resolution certainty, and execution times per pipeline stage.
- **`feature_importances.csv`**: Weights allocated by the ML Stacking Ensemble (RandomForest, ExtraTrees, GradientBoosting) mapping feature significance (e.g., `cos_sim`, `graph_jaccard`, `jaro_winkler`).
- **`adaptive_thresholds.csv`**: Automated Otsu & KDE valley detection logs showing how the system calibrates match decision thresholds dynamically without manual tuning.
- **`audit_trail.csv`**: Complete tracking lineage showing the exact decision (`MERGE` vs `SKIP`) for high-uncertainty entity pairs.
